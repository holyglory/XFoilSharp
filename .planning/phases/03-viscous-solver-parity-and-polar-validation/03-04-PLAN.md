<objective>
Port the viscous/inviscid coupling infrastructure: DIJ influence matrix (QDCALC), stagnation point tracking (STMOVE/STFIND), edge velocity computation (UICALC/QVFUE/GAMQV/QISET/QWCALC), the block-tridiagonal solver (BLSOLV), and a band matrix solver as the third linear solver option.

Purpose: These are the coupling mechanisms that connect the inviscid and boundary layer systems. The DIJ matrix bridges BL displacement to inviscid edge velocity. The block-tridiagonal solver is the linear algebra kernel for the Newton system. The band matrix solver provides the third solver option per CONTEXT.md locked decision ("Three linear solver implementations: custom block-tridiagonal (default), generic sparse LU, band matrix solver"). Without these, the outer Newton loop cannot function.

Output: InfluenceMatrixBuilder, StagnationPointTracker, EdgeVelocityCalculator, BlockTridiagonalSolver, BandMatrixSolver, plus unit tests.

    - DIJ[i,j] represents dUe_i/dSigma_j -- perturbation of source at j changes edge velocity at i
    - DIJ first wake row copies from TE row (matching xpanel.f line 1249 behavior)
    - BuildNumericalDIJ: Produces same-size DIJ by perturbing sigma and re-solving; validates against analytical within 1e-6
    - BLSOLV: Solves 3-equation block-tridiagonal system with mass defect coupling column
    - BLSOLV with VACCEL=0.01: drops small off-diagonal mass coefficients for speed
    - BLSOLV on known 4-station test system: solution matches hand-computed values

  <action>
    **InfluenceMatrixBuilder.cs** (public static class):

    1. **BuildAnalyticalDIJ** -- port of QDCALC from xpanel.f:
       - Uses the already-factored StreamfunctionInfluence (AIJ) matrix from InviscidSolverState
       - For each panel j, constructs RHS from SourceInfluence (BIJ) column j
       - Back-substitutes through the LU-factored AIJ to get delta-gamma from delta-sigma_j
       - DIJ[i,j] = (delta-Ue at i) from unit sigma perturbation at j
       - For wake points: DIJ[wake_k, j] computed from wake velocity influence
       - CRITICAL: Copy TE row to first wake row (xpanel.f line 1249)
       - For LinearVortex: extract from StreamfunctionInfluence + SourceInfluence arrays using LU back-substitution (port of QDCALC logic)

    2. **BuildNumericalDIJ** -- finite-difference validation and HessSmith adapter:
       - For each panel j: perturb sigma[j] by epsilon, re-solve inviscid, measure Ue change
       - DIJ[i,j] = (Ue_perturbed[i] - Ue_original[i]) / epsilon
       - Useful for HessSmith adapter and debugging

    **BlockTridiagonalSolver.cs** (public static class in Numerics/):

    3. **BLSOLV** (xsolve.f:1-488): Solve -- the block-tridiagonal solver for the viscous Newton system.
       Input: ViscousNewtonSystem (VA, VB, VM, VDEL, VZ, ISYS, NSYS)
       Modifies: VDEL in-place (becomes the solution delta vector)

       Key implementation:
       - Forward sweep: Gaussian elimination on 3x3 blocks, reducing VB into VA
       - Mass defect coupling: VM column handled via VACCEL threshold (drop small |VM[k,j,i]| < VACCEL * |VA[k,k,i]|)
       - TE coupling: VZ block couples side 1 and side 2 at TE
       - Back substitution: produces VDEL solution
       - VACCEL = 0.01 default (from AnalysisSettings or hardcoded)

       Port BLSOLV exactly from xsolve.f. The block structure (3x3 diagonal + sub-diagonal + mass column) is deeply coupled to the BL equation formulation. Do NOT reorganize into generic sparse -- the structure IS the performance.

    **BandMatrixSolver.cs** (public static class in Numerics/):

    4. Implement a band-structure solver as the third linear solver option per CONTEXT.md locked decision. This solver reformats the ViscousNewtonSystem's block-tridiagonal + mass-column structure into a banded matrix format and solves using standard band LU factorization:
       - Convert the 3x3 block-tridiagonal structure (VA/VB) plus mass defect coupling (VM) into a single banded matrix with bandwidth determined by the block structure
       - Use standard band LU decomposition and back-substitution
       - Accept the same ViscousNewtonSystem input as BlockTridiagonalSolver
       - Produce the same VDEL solution (within floating-point tolerance)
       - This is NOT the default solver but provides a validation path and alternative when the block-tridiagonal structure assumptions are violated
       - The existing ScaledPivotLuSolver from Phase 2 serves as the "generic sparse LU" second option; BandMatrixSolver is the third

  <action>
    **StagnationPointTracker.cs** (public static class):

    1. **STFIND** (xpanel.f): FindStagnationPoint -- finds the panel node closest to the stagnation point by scanning for minimum |Q| or sign change in surface speed. Returns the stagnation panel index ISP.

    2. **STMOVE** (xpanel.f): MoveStagnationPoint -- when ISP changes during viscous iteration, this:
       - Recomputes BL station mapping (IBLPAN)
       - Shifts BL variables (theta, delta*, etc.) to new station grid
       - Interpolates BL variables at new stations that don't correspond to old ones
       - Updates ITRAN if transition location relative to ISP has changed
       - This is critical for convergence -- fixing ISP causes divergence.

    **EdgeVelocityCalculator.cs** (public static class):

    3. **IBLPAN** (xpanel.f): MapPanelsToBLStations -- creates the mapping from panel node indices to BL station indices. BL station 1 on each side is the stagnation point. Stations increase toward TE. Wake stations follow after IBLTE.

    4. **XICALC** (xpanel.f): ComputeBLArcLength -- computes the arc-length coordinate xi at each BL station from the panel geometry. xi=0 at stagnation point, increases along each surface.

    5. **IBLSYS** (xbl.f): MapStationsToSystemLines -- maps BL station indices to the global Newton system line numbers. Side 1 stations map to lines 1..IBLTE(1)-1, side 2 to lines IBLTE(1)..IBLTE(1)+IBLTE(2)-2, wake to remaining lines. This is the ISYS array.

    6. **UICALC** (xpanel.f): ComputeInviscidEdgeVelocity -- from the panel solution (vortex/source strengths), compute the inviscid edge velocity Ue at each BL station. For the linear-vortex solver, Ue = Q (surface speed from vortex strength).

    7. **QVFUE** (xpanel.f): ComputeViscousEdgeVelocity -- converts the viscous UEDG array to equivalent panel speeds QVIS.

    8. **GAMQV** (xpanel.f): SetVortexFromViscousSpeed -- sets the panel vortex strength GAM from the viscous surface speed QVIS. This is the inverse of UICALC.

    9. **QISET** (xpanel.f): SetInviscidSpeeds -- sets inviscid surface speeds for a given alpha using the basis solutions: Q = Q0*cos(alpha) + Q90*sin(alpha). Used when alpha changes during CL-prescribed iteration.

    10. **QWCALC** (xpanel.f): ComputeWakeVelocities -- computes wake panel velocities from the airfoil TE vorticity and wake geometry.

    Port each function from xpanel.f / xbl.f exactly. The STMOVE routine is particularly critical -- it must correctly interpolate BL variables when the stagnation point shifts.
