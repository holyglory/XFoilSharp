  <action>
    Create PolarSweepRunner as a public static class (or sealed class matching project pattern).

    1. **SweepAlpha** (Type 1 -- SPECAL path):
       ```
       SweepAlpha(
           LinearVortexPanelState panel,
           InviscidSolverState inviscidState,
           AnalysisSettings settings,
           double alphaStartDeg, double alphaEndDeg, double alphaStepDeg)
       -> List<ViscousAnalysisResult>
       ```

       Implementation:
       - Allocate BoundaryLayerSystemState and ViscousNewtonSystem once (reuse across points)
       - For each alpha:
         a. Run inviscid solve at this alpha (LinearVortexInviscidSolver.SolveAtAngleOfAttack)
         b. Pass inviscid result + BL state to ViscousSolverEngine.SolveViscous
         c. On convergence: save BL state as warm-start for next point
         d. On non-convergence: record point with converged=false, try next alpha. Before trying next, attempt warm-start from most recent converged point.
       - **Multi-seed warm-start** (per CONTEXT.md): When a point fails to converge:
         1. Try previous converged point BL as initial guess (already done)
         2. Try other nearby converged points as seeds
         3. Cold-start (re-initialize from inviscid) as last resort
       - Continue sweep even after non-convergence per user decision

    2. **SweepCL** (Type 2 -- SPECCL path):
       ```
       SweepCL(
           LinearVortexPanelState panel,
           InviscidSolverState inviscidState,
           AnalysisSettings settings,
           double clStart, double clEnd, double clStep)
       -> List<ViscousAnalysisResult>
       ```

       Implementation:
       - For each target CL:
         a. Use LinearVortexInviscidSolver.SolveAtLiftCoefficient to get initial alpha
         b. In VISCAL loop, alpha is adjusted each iteration to maintain target CL:
            - After each Newton step, compute viscous CL
            - Adjust alpha: dalpha = (CL_target - CL_viscous) / CL_alpha
            - Recompute inviscid speeds at new alpha (QISET + UICALC)
         c. This is the SPECCL path in Fortran -- alpha is part of the Newton iteration
       - Warm-start with previous converged BL state AND previous converged alpha as initial guess

    3. **SweepRe** (Type 3 -- MRCL type 3):
       ```
       SweepRe(
           LinearVortexPanelState panel,
           InviscidSolverState inviscidState,
           AnalysisSettings baseSettings,
           double fixedCL,
           double reStart, double reEnd, double reStep)
       -> List<ViscousAnalysisResult>
       ```

       Implementation:
       - For each Re:
         a. Create modified AnalysisSettings with new ReynoldsNumber
         b. Use Type 2 (CL-prescribed) logic to find alpha and converge at target CL
         c. The MRCL type 3 relation: Mach and Re may be coupled via REINF1*CL^RETYP (port MRCL from xfoil.f:774)
       - Warm-start across Re values

    4. **Port MRCL** (xfoil.f:774-900): MachReynoldsCoupling -- computes how Mach and Re depend on CL for each polar type. For Type 1: M and Re fixed. For Type 2: M may depend on CL. For Type 3: Re = REINF1 * CL^RETYP.
