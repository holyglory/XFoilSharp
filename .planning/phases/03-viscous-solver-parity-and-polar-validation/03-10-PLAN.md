    ```
    // Pre-iteration setup
    1. Compute ISYS mapping: var (isysMap, nsys) = EdgeVelocityCalculator.MapStationsToSystemLines(iblte, nbl)
    2. Create Newton system: var newtonSystem = new ViscousNewtonSystem(nsys, nWake)
       newtonSystem.SetupISYS(isysMap, nsys)
    3. Build DIJ matrix: var dij = InfluenceMatrixBuilder.BuildAnalyticalDIJ(inviscidState, panel, nWake)
    4. Compute compressibility parameters (COMSET):
       tkbl = PanelGeometryBuilder.ComputeCompressibilityParameters(settings.MachNumber)
       or compute manually: tkbl = M^2 / (1 + sqrt(1-M^2))^2 for Karman-Tsien
       For M=0: tkbl=0, qinfbl=qinf, hstinv=0, rstbl=1, reybl=reinf
    5. Initialize BL from inviscid Ue (keep existing InitializeBLFromInviscidUe but
       call TransitionModel for initial transition location instead of simplified model)

    // Newton iteration loop (matching VISCAL)
    for (int iter = 0; iter < maxIter; iter++)
    {
        // a. Build Newton system (SETBL)
        double rmsbl = ViscousNewtonAssembler.BuildNewtonSystem(
            blState, newtonSystem, dij, settings,
            isAlphaPrescribed: true, wakeGap, isp, n,
            tkbl, qinfbl, tkbl_ms,
            hstinv, hstinv_ms,
            rstbl, rstbl_ms,
            reybl, reybl_re, reybl_ms, hvrat);

        // b. Solve block-tridiagonal system (BLSOLV)
        BlockTridiagonalSolver.Solve(newtonSystem, vaccel: 0.01);

        // c. Apply Newton update with relaxation (UPDATE)
        var (rlx, updatedRms, newTrustRadius, accepted) =
            ViscousNewtonUpdater.ApplyNewtonUpdate(
                blState, newtonSystem, settings.ViscousSolverMode,
                hstinv, wakeGap, trustRadius, prevRmsbl, rmsbl);
        trustRadius = newTrustRadius;
        prevRmsbl = rmsbl;

        // d. Update edge velocities from mass defect (QVFUE + GAMQV)
        //    After the Newton update, UEDG has been modified by the updater.
        //    Convert back to panel speeds for the next iteration.

        // e. Relocate stagnation point if it has moved (STMOVE)
        int newIsp = FindStagnationPointByMinSpeed(
            EdgeVelocityCalculator.ConvertUedgToSpeeds(blState, isp, n), n);
        if (newIsp != isp)
        {
            StagnationPointTracker.MoveStagnationPoint(blState, isp, newIsp, ...);
            isp = newIsp;
            // Rebuild ISYS, IBLPAN after stagnation move
        }

        // f. Compute CL, CD, CM from current viscous solution
        //    CL: from CLCALC using viscous QVIS speeds
        //    CD: from DragCalculator
        //    CM: from moment integration with viscous speeds
        cl = ComputeViscousCL(blState, panel, inviscidState, alphaRadians);
        cd = DragCalculator.ComputeDrag(...).CD;
        cm = ComputeViscousCM(blState, panel, inviscidState, alphaRadians);

        // Record convergence history
        convergenceHistory.Add(new ViscousConvergenceInfo { ... });

        // g. Check convergence: rmsbl < tolerance (XFoil uses 1e-4 for RLXBL mode)
        if (rmsbl < tolerance)
        {
            converged = true;
            break;
        }
    }
    ```

    **Key changes from the Picard approach:**
    - Remove MarchBoundaryLayer entirely (the Newton system handles BL equations)
    - Remove UpdateEdgeVelocityCarterCoupling entirely (Newton handles Ue coupling via DIJ)
    - Remove inline ComputeInstabilityOnset/ComputeAmplificationGrowthRate (TransitionModel.CheckTransition is called from within BuildNewtonSystem via plan 09)
    - Keep FindStagnationPointByMinSpeed (still needed for STMOVE)
    - Keep InitializeBLFromInviscidUe for the initial BL state (before first Newton iteration)
    - Keep EstimateDrag as a fallback but prefer DragCalculator for final result

    **Add CL/CM from viscous speeds.** XFoil computes CL and CM from the viscous panel speeds (QVIS), not the inviscid speeds. After Newton convergence, compute QVIS from UEDG via QVFUE, then compute CL via CLCALC (already ported as LinearVortexInviscidSolver.ComputeLiftCoefficient -- but need to pass viscous gamma). The key insight: viscous CL != inviscid CL because the effective circulation changes with displacement thickness.

    Specifically, add a `ComputeViscousCL` helper that:
    1. Converts blState.UEDG back to panel speeds (QVFUE: qvis[i] = UEDG[ibl, side] * VTI)
    2. Computes gamma from qvis (GAMQV: gamma = qvis for linear-vortex)
    3. Calls the lift coefficient integration using the viscous gamma

    Similarly for CM.

    **Keep the existing ViscousSolverEngineTests** but update tolerance expectations. The NACA 0012 Re=1e6 tests should now produce CL and CD much closer to XFoil reference values. Update the test assertions to reflect Newton-level accuracy instead of Picard-level.
