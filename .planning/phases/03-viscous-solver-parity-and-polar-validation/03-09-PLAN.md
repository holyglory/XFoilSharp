<objective>
Fix the Newton system infrastructure so that ViscousNewtonAssembler (SETBL), BlockTridiagonalSolver (BLSOLV), and ViscousNewtonUpdater (UPDATE) work correctly together. Purpose: Unblock the Newton coupling loop (plan 10) by making the individual components correct.
Output: Fixed ViscousNewtonSystem, ViscousNewtonAssembler, ViscousNewtonUpdater, BlockTridiagonalSolver, with regression tests.
</objective>

    **Fix ViscousNewtonAssembler.BuildNewtonSystem:**

    1. **Use global system line IV for all array indexing.** Replace `va[k, 0, ibl]` with `va[k, 0, iv]` throughout. The loop already computes `iv` from ISYS lookup -- use it consistently for ALL array writes (VA, VB, VDEL, VM).

    2. **Fix DIJ source-to-mass coupling.** The current code has:
       ```
       double ue_m = -vtiI * vtiJ * dij[iRow, jCol];
       ```
       This is wrong because DIJ gives dUe/dSigma (source strength), but the Newton variable is mass defect (m = delta* * Ue), not sigma. The correct mapping from XFoil (xbl.f SETBL lines ~400-450) is:
       - `U2_M[jv] = -VTI[iv] * VTI[jv] * DIJ[IPAN[iv], IPAN[jv]]` where IPAN maps BL station to panel index
       - The VTI sign factor is `+1` on side 0 (upper), `-1` on side 1 (lower), `+1` on wake
       - The panel index mapping: for side 0, `IPAN[ibl] = ISP - ibl`; for side 1, `IPAN[ibl] = ISP + ibl`; for wake, `IPAN[ibl] = N + (ibl - IBLTE[1])`

       Add a `GetPanelIndex(ibl, side, isp, n, blState)` helper that returns the panel node index for a given BL station. Use this for DIJ lookup instead of the current `GetDijIndex` which has a simplified linear offset.

       The key fix: DIJ[i,j] maps sigma perturbation at panel j to Ue change at panel i. The mass defect change dM at BL station jv corresponds to source change dSigma = VTI[jv] * dM / (panel spacing). The VTI factor accounts for the direction of the normal velocity relative to the BL convention.

       Replace the VM assembly with:
       ```csharp
       for (int jv = 0; jv < nsys; jv++)
       {
           int jbl = isys[jv, 0];
           int jside = isys[jv, 1];
           int iPan = GetPanelIndex(ibl, side, isp, n, blState);
           int jPan = GetPanelIndex(jbl, jside, isp, n, blState);
           double vtiI = GetVTI(ibl, side, blState);
           double vtiJ = GetVTI(jbl, jside, blState);
           double ue_mj = -vtiI * vtiJ * dij[iPan, jPan];
           // d2_m contribution: d(delta*)/d(mass) at station jv via Ue change
           double d2_mj = d2_u2 * ue_mj * u2_uei;
           if (jv == iv) d2_mj += d2_m2; // self-coupling
           for (int k = 0; k < 3; k++)
               vm[k, jv, iv] = localResult.VS2[k, 2] * d2_mj + localResult.VS2[k, 3] * ue_mj * u2_uei;
       }
       ```
       Note the `u2_uei` factor that was missing -- the VS2 sensitivities are w.r.t. compressible U2, but DIJ gives dUe_incompressible. Must chain through ConvertToCompressible.

    3. **Add `isp` and `n` parameters** to BuildNewtonSystem (stagnation point index and total panel count) for the GetPanelIndex helper.

    4. **Wire TransitionModel.CheckTransition.** In the BL march loop inside BuildNewtonSystem, when `!wake && !turb` and `ibl > 1`:
       - Call `TransitionModel.CheckTransition(x1, xsi, u1, uei, t1, thi, d1, dsi, s1, cti, dw1, dswaki, ampl1, ami, hk1, hk2, rt1, rt2, ncrit, ...)` using the actual station-pair variables
       - If `result.TransitionOccurred`, update `blState.ITRAN[side] = ibl` and set `tran = true`
       - Pass the TRCHEK2 Jacobian sensitivities into the BL system assembly so the Newton system accounts for transition location sensitivity

    **Fix ViscousNewtonUpdater.ApplyNewtonUpdate:**

    1. **Re-index VDEL reads.** The updater currently reads `vdel[k, 0, ibl]` -- change to `vdel[k, 0, iv]` where iv is the global system line. Use the ISYS mapping in the Newton system to iterate over (iv -> ibl, side) pairs.

    2. **Fix Ue update from mass coupling.** After the tridiagonal solve, VDEL contains deltas for (Ctau/ampl, theta, mass). The Ue change comes from the mass defect change via the DIJ coupling:
       `dUe[iv] = sum_jv( -VTI[iv]*VTI[jv]*DIJ[iPan_iv, iPan_jv] * dMass[jv] )`
       This must be computed in the update step and applied to UEDG. The current code has `duedg = 0.0` which means Ue is never updated from the Newton step -- this is the second root cause of the O(1e7) issue (the correction was being computed but never applied).

    Ensure all existing tests compile (update any test that constructs ViscousNewtonSystem with the old API).
