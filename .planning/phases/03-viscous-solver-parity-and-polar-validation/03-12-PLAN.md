**ViscousSolverEngine.cs** -- Add `TextWriter? debugWriter = null` parameter to SolveViscousFromInviscid:
- Before each iteration: `debugWriter?.WriteLine($"=== ITER {iter} ===");`
- After Newton update or BL march: `debugWriter?.WriteLine($"POST_UPDATE RMSBL= {marchRms:E15.8} RMXBL= {marchRms*2:E15.8} RLX= {rlx:E15.8}");`
- After CL/CD/CM computation: `debugWriter?.WriteLine($"POST_CALC CL= {cl:E15.8} CD= {cd:E15.8} CM= {cm:E15.8}");`
- On convergence: `debugWriter?.WriteLine($"CONVERGED iter= {iter}");`
- Pass debugWriter through to BuildNewtonSystem, Solve, and ApplyNewtonUpdate.

**ViscousNewtonAssembler.cs** -- Add `TextWriter? debugWriter = null` parameter to BuildNewtonSystem:
Inside the IBL loop, after the system coefficients are written to VA/VB/VM/VDEL, add:
```csharp
if (debugWriter != null)
{
    debugWriter.WriteLine($"STATION IS={side+1,2} IBL={ibl,4} IV={iv+1,4}");
    debugWriter.WriteLine($"BL_STATE x={xsi:E15.8} Ue={uei:E15.8} th={thi:E15.8} ds={dsi:E15.8} m={mdi:E15.8}");
    debugWriter.WriteLine($"VA_ROW1{va[0,0,iv]:E15.8}{va[0,1,iv]:E15.8}");
    debugWriter.WriteLine($"VA_ROW2{va[1,0,iv]:E15.8}{va[1,1,iv]:E15.8}");
    debugWriter.WriteLine($"VA_ROW3{va[2,0,iv]:E15.8}{va[2,1,iv]:E15.8}");
    debugWriter.WriteLine($"VB_ROW1{vb[0,0,iv]:E15.8}{vb[0,1,iv]:E15.8}");
    debugWriter.WriteLine($"VB_ROW2{vb[1,0,iv]:E15.8}{vb[1,1,iv]:E15.8}");
    debugWriter.WriteLine($"VB_ROW3{vb[2,0,iv]:E15.8}{vb[2,1,iv]:E15.8}");
    debugWriter.WriteLine($"VDEL_R{vdel[0,0,iv]:E15.8}{vdel[1,0,iv]:E15.8}{vdel[2,0,iv]:E15.8}");
    debugWriter.WriteLine($"VDEL_S{vdel[0,1,iv]:E15.8}{vdel[1,1,iv]:E15.8}{vdel[2,1,iv]:E15.8}");
    debugWriter.WriteLine($"VSREZ{vsrez0:E15.8}{vsrez1:E15.8}{vsrez2:E15.8}");
}
```

CRITICAL FORMAT NOTE: The Fortran format uses 1-based indexing (IS=1,2; IBL starts at 2; IV starts at 1). The C# uses 0-based arrays. The debug output MUST use Fortran conventions (add 1 to side, add 1 to iv) so lines match:
- `side+1` for IS
- `ibl` already matches if the C# loop starts at ibl=2
- `iv+1` for IV (C# is 0-based, Fortran is 1-based)

Also log VSREZ (the residual vector from BLSYS call). These are the local BL equation residuals before being stuffed into VDEL. The C# equivalent comes from BoundaryLayerSystemAssembler.

Also log DUE2 "forced change": `debugWriter?.WriteLine($"DUE2= {due2:E15.8} DDS2= {dds2:E15.8}");`

Also log transition: when transition detected, `debugWriter?.WriteLine($"TRANSITION IS={side+1} IBL={ibl} XT={xt:E15.8}");`
