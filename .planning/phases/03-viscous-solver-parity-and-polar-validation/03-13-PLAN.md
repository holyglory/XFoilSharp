<interfaces>
From BoundaryLayerSystemAssembler.cs (the VS1/VS2/VSREZ output that feeds into VA/VB/VM/VDEL):
The assembler produces a local 4x5 system (VS1[4,5], VS2[4,5], VSREZ[4], VSR[4], VSM[4], VSX[4])
which SETBL (ViscousNewtonAssembler) stuffs into the global Newton system blocks.

The key chain is:
1. BoundaryLayerCorrelations.cs computes BL correlations (Cf, Hk, Hs, etc.) with Jacobian sensitivities
2. BoundaryLayerSystemAssembler.AssembleLocalSystem calls BLSYS/BLDIF/TRDIF to build VS1/VS2/VSREZ
3. ViscousNewtonAssembler.BuildNewtonSystem stuffs VS1/VS2/VSREZ into global VA/VB/VM/VDEL arrays
4. BlockTridiagonalSolver.Solve performs Gaussian elimination on VA/VB/VM to get VDEL solutions
5. ViscousNewtonUpdater.ApplyNewtonUpdate reads VDEL solutions and applies Newton deltas

The divergence could be at any of steps 2-5. The comparison script will narrow it down.
</interfaces>

```python
#!/usr/bin/env python3
"""Compare Fortran and C# intermediate value dumps to find divergence."""
```

The script should:

1. Parse both dump files into structured data:
   - Group by iteration (=== ITER N ===)
   - Within each iteration, group by station (STATION IS=X IBL=Y IV=Z)
   - Extract tagged values: BL_STATE, VA_ROW1-3, VB_ROW1-3, VDEL_R, VDEL_S, VSREZ, DUE2
   - Extract per-iteration: POST_UPDATE (RMSBL, RLX), POST_CALC (CL, CD, CM)
   - Extract VDEL_FWD and VDEL_SOL from BLSOLV

2. Compare iteration 1 first (this is where we expect to see the initial divergence):
   - For each station, compare BL_STATE values (x, Ue, theta, dstar, mass) -- these should match closely since they come from initialization
   - Compare VA_ROW1-3 and VB_ROW1-3 blocks (these come from SETBL stuffing VS1/VS2)
   - Compare VDEL_R (residual) and VDEL_S (sensitivity) values
   - Compare VSREZ (local equation residuals)

3. For each comparison, compute relative error: `|fortran - csharp| / max(|fortran|, 1e-20)`

4. Report the FIRST station where any value exceeds 1% relative error, showing:
   - Which iteration, which side, which station
   - Which value diverged (e.g., "VA_ROW1[0] = Fortran 1.23e-3 vs C# 4.56e-3, rel_err=270%")
   - Context: the BL_STATE at that station (both Fortran and C# values)
   - The previous station's values (to see if divergence propagated or originated here)

5. Also report a summary table:
   - Per-iteration: RMSBL comparison, CL/CD comparison
   - First divergence point per category (BL_STATE, VA, VB, VDEL, VSREZ)


**Pattern A: BL_STATE matches but VA/VB diverge** -> Problem is in ViscousNewtonAssembler's stuffing of VS1/VS2 into VA/VB (the D2_M, U2_M, D1_M, U1_M sensitivity chains, or the VM column computation).
