<objective>
Instrument Fortran XFoil with logging in SETBL, BLSOLV, UPDATE, and VISCAL to dump intermediate values at each iteration and station. Build the instrumented binary and run the reference case (NACA 0012, Re=1e6, alpha=0) to capture a structured dump file for comparison with the C# implementation.

Purpose: The Newton Jacobians in the C# BoundaryLayerSystemAssembler diverge end-to-end. To find the root cause, we need a line-by-line comparison of intermediate values between Fortran XFoil and the C# port. This plan creates the Fortran reference data.

Output: Instrumented Fortran source files, build script, run script, and reference_dump.txt containing all intermediate values needed for comparison.
</objective>

**xoper_debug.f** -- Instrument VISCAL (lines 2583-2729):
- At start of VISCAL: OPEN(50, FILE='debug_dump.txt', STATUS='REPLACE')
- Before each SETBL call (line 2664): WRITE(50,*) '=== ITER', ITER, '==='
- After SETBL returns: WRITE(50,*) 'POST_SETBL RMSBL=', RMSBL (note: RMSBL isn't set until UPDATE, so log this AFTER UPDATE)
- After UPDATE returns (line 2670): WRITE(50,*) 'POST_UPDATE RMSBL=', RMSBL, 'RMXBL=', RMXBL, 'RLX=', RLX
- After CLCALC/CDCALC (line 2694): WRITE(50,*) 'POST_CALC CL=', CL, 'CD=', CD, 'CM=', CM, 'CDF=', CDF
- At convergence (line 2705): WRITE(50,*) 'CONVERGED iter=', ITER
- At end of VISCAL: CLOSE(50)

Create two shell scripts and run them:

**build_debug.sh**: Builds the instrumented XFoil binary.
- Create a temporary build directory: tools/fortran-debug/build/
- Copy ALL source from f_xfoil/src/ into tools/fortran-debug/build/
- Overwrite xbl.f, xoper.f, xsolve.f with the instrumented versions from Task 1
- Compile with gfortran using the same flags as the cmake build: `-std=legacy -O2`
- The XFoil build links against the plotlib library. Since we do not need plotting for batch mode, stub out the plot calls. The simplest approach: compile with the existing plotlib from f_xfoil/build/plotlib/ (libplt.a) if it exists, OR create a minimal stub library that provides the required symbols (PLOPEN, PLCLOS, etc. as no-op subroutines).
- Produce tools/fortran-debug/build/xfoil_debug binary.
- Handle the fact that XFoil needs the ORRS data: set environment or just ensure we are running from f_xfoil/ directory.

If the plotlib approach is too complex, an alternative: use gfortran to compile only the core subroutines into a minimal driver program that:
1. Reads a NACA 0012 coordinate file
2. Calls the panel, inviscid, and viscous solvers
3. Does NOT require plotting

However, XFoil's subroutines are deeply interconnected via COMMON blocks. The most reliable approach is to compile the full XFoil with the instrumented files replacing the originals.

**run_reference.sh**: Runs the instrumented binary on the reference case.
- Changes to tools/fortran-debug/build/ directory
- Creates a piped input for XFoil's interactive commands:
```
NACA 0012
PANE
OPER
VISC 1000000
ITER 20
ALFA 0

QUIT
```
- Pipes this input to ./xfoil_debug
- The WRITE(50,...) statements will create debug_dump.txt in the working directory
- Copies debug_dump.txt to tools/fortran-debug/reference_dump.txt
- Validates the dump file is non-empty and contains expected markers (STATION, POST_UPDATE, CONVERGED)

Execute both scripts:
```bash
chmod +x tools/fortran-debug/build_debug.sh
chmod +x tools/fortran-debug/run_reference.sh
bash tools/fortran-debug/build_debug.sh
bash tools/fortran-debug/run_reference.sh
```
