**XFoilPanelState.cs**: Holds panel geometry matching XFOIL.INC COMMON/CR05/ and COMMON/CR06/ layout.
- Use 1-based indexing: allocate arrays of size N+2, ignore index 0 (per research Pitfall 1). Add a comment at the top explaining this convention.
- Fields (all public, mutable):
  - `int N` -- number of panel nodes (airfoil only, not wake)
  - `double[] X, Y` -- node coordinates, size IZX (use 600 as default max, matching IQX=360 + IWX)
  - `double[] S` -- arc length parameter
  - `double[] XP, YP` -- spline derivatives dX/dS, dY/dS
  - `double[] NX, NY` -- outward normal vectors at each node
  - `double[] APANEL` -- panel angle at each node
  - `double SLE, XLE, YLE` -- leading edge arc length, coordinates
  - `double XTE, YTE` -- trailing edge coordinates
  - `double CHORD` -- chord length
- Constructor takes maxSize (default 600), allocates all arrays.
- Keep the existing Panel/PanelMesh classes untouched (they serve the Hess-Smith solver).

**XFoilInviscidState.cs**: Holds inviscid solution arrays matching COMMON/CR03/, COMMON/CR06/, COMMON/QMAT/.
- Fields (all public, mutable):
  - `double[] GAM` -- vortex strength at each node, size IQX (360)
  - `double[,] GAMU` -- two basis vortex distributions, size [IQX, 2] (1-based: use [IQX+1, 3])
  - `double[] GAM_A` -- dGAM/dAlpha
  - `double[] SIG` -- source strength, size IZX
  - `double[,] AIJ` -- streamfunction influence matrix, size [IQX, IQX] (1-based)
  - `double[,] BIJ` -- source influence on streamfunction, size [IQX, IZX] (1-based)
  - `int[] AIJPIV` -- pivot indices for LUDCMP
  - `double[] QINV` -- inviscid surface speed, size IZX
  - `double[,] QINVU` -- two basis surface speeds, size [IZX, 2] (1-based)
  - `double[] QINV_A` -- dQINV/dAlpha
  - `double PSIO` -- internal streamfunction value
  - `double GAMTE, GAMTE_A` -- TE vortex strength and its alpha derivative
  - `double SIGTE, SIGTE_A` -- TE source strength and its alpha derivative
  - `double DSTE` -- TE gap magnitude
  - `double ANTE, ASTE` -- TE gap angle components
  - `bool SHARP` -- true if TE gap < 0.0001*chord
  - `double SST, SST_GO, SST_GP` -- stagnation point arc length and derivatives
  - `bool LGAMU, LQAIJ` -- flags: basis solutions computed, AIJ factored
  - Workspace arrays from COMMON/QMAT/: `double[] DZDG, DZDN, DZDM, DQDG, DQDM` (used by PSILIN)
  - `double QTAN1, QTAN2` -- tangential velocity at TE nodes
  - `double Z_QINF, Z_ALFA` -- streamfunction sensitivities
- Constructor takes maxN (default 360), maxNW (default 47, matching IWX), allocates all arrays with 1-based sizing.

**XFoilInviscidResult.cs**: Lightweight result container (immutable).
- Properties: `double CL, CM, CDP, CL_ALF, CL_MSQ`
- `double[] CPI` -- pressure coefficient distribution (inviscid)
- Constructor takes all values.

Port spline routines as static methods in a new `XFoilSpline` class. All use 1-based indexing.

**TRISOL** (spline.f lines 185-218): Standard Thomas algorithm for tridiagonal systems. Method: `Trisol(double[] a, double[] b, double[] c, double[] d, int kk)`. Arrays are 1-based, kk is count.

**SPLINE** (spline.f lines 21-60): Cubic spline with zero second derivative end conditions. Method: `Spline(double[] x, double[] xs, double[] s, int n)`. Computes xs[] (derivatives) in-place. Uses TRISOL internally.

**SPLIND** (spline.f lines 63-130): Cubic spline with three BC modes based on xs1/xs2 values:
- xs1/xs2 = -999.0: zero third derivative end condition (used by SEGSPL for XFoil's default)
- xs1/xs2 = -998.0: zero second derivative end condition
- otherwise: specified first derivative end condition
Method: `Splind(double[] x, double[] xs, double[] s, int n, double xs1, double xs2)`.
Port the three-way BC branching exactly as written in Fortran.

**SEGSPL** (spline.f lines 533-558): Segmented spline allowing derivative discontinuities at corners. Detects segment breaks where arc-length spacing ratio exceeds 3x, then calls SPLIND with -999.0 BCs for each segment independently. Method: `Segspl(double[] x, double[] xs, double[] s, int n)`.

**SEVAL** (spline.f lines 218-244): Cubic spline evaluation at arbitrary parameter. Method: `Seval(double ss, double[] x, double[] xs, double[] s, int n)`. Uses binary search for interval location.

**DEVAL** (spline.f lines 245-272): Cubic spline derivative evaluation. Method: `Deval(double ss, double[] x, double[] xs, double[] s, int n)`.

Also port **SCALC** (spline.f lines 436-451): Arc-length parameter computation. Method: `Scalc(double[] x, double[] y, double[] s, int n)`. Computes cumulative arc length s[i] from (x,y) coordinates.

Preserve exact Fortran operation ordering. Add line-number comments. Do NOT modify the existing CubicSpline.cs.
