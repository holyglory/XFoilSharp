**Distribute**: Takes raw airfoil coordinates and produces a panel node distribution.

Signature:
```csharp
public static void Distribute(
    double[] inputX, double[] inputY, int inputCount,
    LinearVortexPanelState panel,
    int desiredNodeCount = 160,
    double curvatureWeight = 1.0,
    double trailingEdgeDensityRatio = 0.15,
    double curvatureDensityRatio = 0.2)
```

Parameters:
- inputX/Y: raw airfoil coordinates from a file or generator (0-based, inputCount points)
- panel: output LinearVortexPanelState to populate
- desiredNodeCount: target N (default 160, matching XFoil's NPAN default)
- curvatureWeight: CVPAR -- controls how strongly curvature affects panel density (1.0 = XFoil default)
- trailingEdgeDensityRatio: CTERAT -- ratio of TE panel density to average (0.15 = XFoil default)
- curvatureDensityRatio: CTRRAT -- additional curvature-based clustering strength (0.2 = XFoil default)

**Algorithm (implement cleanly, not as Fortran transliteration):**

1. **Spline the input coordinates**: Compute arc-length parameterization of input points. Fit parametric cubic splines to X(s) and Y(s) using zero-third-derivative boundary conditions (this is XFoil's SPLIND with the -999 BC mode, which is `SplineBoundaryCondition.ZeroThirdDerivative` from Plan 01).

2. **Find the leading edge**: The LE is the point on the splined airfoil where X is minimized. Use Newton iteration: start from an initial guess (the input point with minimum X), then iterate s = s - dX/ds / (d2X/ds2) until |dX/ds| < tolerance. Store as LeadingEdgeArcLength, LeadingEdgeX, LeadingEdgeY.

3. **Normalize to [0,1] parameter**: Map the arc-length parameter to a normalized parameter where 0 = TE (upper surface), 0.5 = LE, 1.0 = TE (lower surface). Let sTotal = total arc length. Upper surface runs from s=0 to s=sLE, lower surface from s=sLE to s=sTotal. Normalize: for upper surface, t = 0.5 * s/sLE; for lower surface, t = 0.5 + 0.5 * (s - sLE)/(sTotal - sLE).

4. **Base cosine distribution**: Create desiredNodeCount base positions using cosine spacing: t[i] = 0.5 * (1 - cos(pi * i / (N-1))) for i = 0..N-1. This naturally clusters near t=0 (TE) and t=0.5 (LE).

5. **Curvature-based refinement** (iterative): Refine the base distribution to cluster panels where curvature is high:
   - Compute curvature at each base position: kappa = |x'*y'' - y'*x''| / (x'^2 + y'^2)^1.5
   - Compute a density function: density[i] = 1 + curvatureWeight * (kappa[i] / kappaMax)
   - Also increase density near TE by factor related to trailingEdgeDensityRatio
   - Redistribute nodes proportional to the density function (integrate density, invert the CDF)
   - Iterate this refinement 2-3 times for convergence

6. **Evaluate coordinates at new positions**: Map the refined t[] values back to arc-length parameter s values. Evaluate the splined X(s) and Y(s) at each position to get final node coordinates.

7. **Populate panel state**:
   - Set panel.Resize(desiredNodeCount)
   - Copy coordinates to panel.X[], panel.Y[]
   - Compute arc-length via ParametricSpline.ComputeArcLength
   - Fit splines for XDerivative, YDerivative via ParametricSpline.FitWithBoundaryConditions using zero-third-derivative BCs
   - Set panel.TrailingEdgeX/Y = midpoint of node 0 and node N-1
   - Set panel.LeadingEdgeX/Y/ArcLength from LE detection
   - Set panel.Chord = distance from LE to TE

**Helper methods to implement:**

- `FindLeadingEdge(...)`: Newton iteration to find minimum-X point on splined airfoil
- `ComputeCurvature(...)`: Curvature from spline derivatives at a given parameter
- `RedistributeByDensity(...)`: Redistributes node parameters proportional to a density function

Also implement arc-length re-integration (XFoil's SPLNXY algorithm): after distributing nodes, re-compute a true arc-length parameterization (not just cumulative chord) by integrating sqrt(dx^2+dy^2) along the spline between consecutive nodes. This gives more accurate arc-length values than simple point-to-point distances.
