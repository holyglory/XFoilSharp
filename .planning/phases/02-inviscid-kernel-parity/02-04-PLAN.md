**LinearVortexInviscidSolver.cs**: New class in Services. Implements the complete inviscid solver pipeline using the linear-vorticity streamfunction formulation. This implements the algorithms from XFoil's GGCALC, SPECAL, SPECCL, QISET, CLCALC, CPCALC routines in clean idiomatic C#.

Methods:

**AssembleAndFactorSystem(LinearVortexPanelState panel, InviscidSolverState state, double freestreamSpeed)**:
Implements the GGCALC algorithm -- assembles and solves the streamfunction influence system.
1. Ensure geometry is prepared: call PanelGeometryBuilder.ComputeNormals, ComputePanelAngles, ComputeTrailingEdgeGeometry if not already done.
2. For each airfoil node i = 0..N-1:
   - Call StreamfunctionInfluenceCalculator.ComputeInfluenceAt(i, X[i], Y[i], NX[i], NY[i], false, true, panel, state, freestreamSpeed, 0)
   - Copy sensitivities into influence matrix: StreamfunctionInfluence[i,j] = StreamfunctionVortexSensitivity[j] for j=0..N-1
   - Copy source influence: SourceInfluence[i,j] = -StreamfunctionSourceSensitivity[j] for j=0..N-1
   - Set StreamfunctionInfluence[i, N] = -1.0 (internal streamfunction column)
   - Set basis RHS: BasisVortexStrength[i, 0] = -freestreamSpeed * Y[i], BasisVortexStrength[i, 1] = freestreamSpeed * X[i]
3. Kutta condition (row N): StreamfunctionInfluence[N, 0] = 1.0, StreamfunctionInfluence[N, N-1] = 1.0 (gamma at first + last node = 0). Zero all other entries in row N.
4. **Sharp TE override**: If IsSharpTrailingEdge, replace the row for the last airfoil node (row N-1) with an internal bisector zero-velocity condition:
   - Compute bisector angle from TE node tangent directions
   - Place control point slightly inside the TE
   - Call ComputeInfluenceAt at that bisector point to get velocity sensitivities
   - Set StreamfunctionInfluence[N-1, j] = VelocityVortexSensitivity[j]
   - Adjust basis RHS accordingly (cosine/sine of bisector angle components)
5. LU-factor: ScaledPivotLuSolver.Decompose(StreamfunctionInfluence, PivotIndices, N+1)
6. Solve both basis RHS vectors: extract each column, BackSubstitute, store back.
7. Copy basis surface speeds: BasisInviscidSpeed[i, 0/1] from BasisVortexStrength
8. Set AreBasisSolutionsComputed = true, IsInfluenceMatrixFactored = true

**SolveAtAngleOfAttack(double alphaRadians, LinearVortexPanelState panel, InviscidSolverState state, double freestreamSpeed, double machNumber)** -> LinearVortexInviscidResult:
Implements the SPECAL algorithm.
1. If !AreBasisSolutionsComputed: call AssembleAndFactorSystem first.
2. Superimpose basis solutions: VortexStrength[i] = cos(alpha)*BasisVortexStrength[i,0] + sin(alpha)*BasisVortexStrength[i,1]
3. Compute VortexStrengthAlphaDerivative[i] = -sin(alpha)*Basis[i,0] + cos(alpha)*Basis[i,1]
4. InternalStreamfunction = VortexStrength[N] (the N+1th entry, which is the internal psi value)
5. Update TE vortex/source strengths from TE geometry ratios
6. Compute inviscid speed: InviscidSpeed[i] = cos(alpha)*BasisInviscidSpeed[i,0] + sin(alpha)*BasisInviscidSpeed[i,1]
7. Newton iteration on CL-Mach coupling (for M=0, this converges in 1 iteration -- just call IntegratePressureForces directly)
8. Return result from IntegratePressureForces

**SolveAtLiftCoefficient(double targetCl, LinearVortexPanelState panel, InviscidSolverState state, double freestreamSpeed, double machNumber)** -> LinearVortexInviscidResult:
Implements the SPECCL algorithm -- Newton iteration to find alpha that produces desired CL.
1. Initial guess: alpha = targetCl / (2 * PI)
2. Loop: call SolveAtAngleOfAttack(alpha), use CL_alpha derivative to update alpha = alpha + (targetCl - CL) / CL_alpha
3. Converge when |CL - targetCl| < 1e-6

**ComputeInviscidSpeed(double alphaRadians, InviscidSolverState state)**:
Simple superposition: Q[i] = cos(alpha)*QBasis[i,0] + sin(alpha)*QBasis[i,1]

**IntegratePressureForces(LinearVortexPanelState panel, InviscidSolverState state, double alphaRadians, double machNumber, double freestreamSpeed, double momentRefX, double momentRefY)** -> LinearVortexInviscidResult:
Implements the CLCALC algorithm -- integrates surface pressures.
1. Compute compressibility: beta = sqrt(1 - M^2), bFac = M^2 / (2*(1+beta))
2. Compute Karman-Tsien corrected Cp at each node: cpIncompressible = 1 - (Q/Qinf)^2, cpCorrected = cpIncompressible / (beta + bFac*cpIncompressible)
3. For each panel (between node i and i+1, wrapping N-1 back to 0):
   - DX = projected chord increment in wind axis (rotated by alpha)
   - DY = projected thickness increment in wind axis
   - averageCp = 0.5 * (Cp[i+1] + Cp[i])
   - deltaCp = Cp[i+1] - Cp[i]
   - CL += DX * averageCp
   - CDP -= DY * averageCp
   - **CM -= DX*(averageCp*armX + deltaCp*DX/12.0) + DY*(averageCp*armY + deltaCp*DY/12.0)** -- NOTE the DG*DX/12 and DG*DY/12 second-order correction terms. This is critical for CM accuracy.
4. Compute CL_alpha and CL_MachSquared analytically alongside the integration (using the gamma-alpha derivatives)
5. Return LinearVortexInviscidResult

**ComputePressureCoefficients(double[] surfaceSpeed, double freestreamSpeed, double machNumber, double[] pressureCoefficients, int count)**:
Karman-Tsien corrected Cp from surface speed.

**Tolerance strategy for aerodynamic correctness (NOT Fortran parity):**
- CL near zero (alpha=0, symmetric): absolute tolerance 1e-6
- CL at moderate alpha: relative tolerance within 5% of thin-airfoil theory. Thin-airfoil theory gives CL = 2*pi*alpha (radians) for flat plates; real airfoils with thickness are slightly higher. Accept CL in range [0.9*theory, 1.1*theory].
- CM for symmetric airfoil at quarter-chord: |CM| < 0.02 (should be near zero)
- CM for cambered airfoil: correct sign (negative for positive camber about quarter-chord)
- CL linearity: for two alpha values, CL difference / alpha difference should be close to 2*pi (~6.28), within 10%
- Panel independence: CL at 100 panels vs 200 panels within 1% relative agreement
