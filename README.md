# XFoilSharp

A managed C# port of XFoil 6.97 (MIT Drela's airfoil analysis tool). Modern, testable, headless-first implementation that keeps discrepancies with the original Fortran visible and documented.

## The three branches

The codebase carries three parallel implementations of the solver stack, organised on a single improvement path:

```
#1 binary parity (float) → #2 algorithmic parity (double) → #3 modern
```

Anything off this path is dead weight. The Phase 3 cleanup deleted the alternative inviscid path (Hess-Smith) and the surrogate viscous shims so only this line remains.

### #1 Binary parity (float)

The reference branch. Single-precision-float code that reproduces Fortran XFoil 6.97 bit-exactly: same algorithm, same call order, same arithmetic rounding under `gfortran -O0 -ffp-contract=off`. The Fortran kernel used `REAL` (32-bit) intermediates, so the parity branch keeps `float`/`MathF` throughout.

- **Lives in**: `src/XFoil.Solver/Services/*.cs`, `src/XFoil.Solver/Numerics/*.cs` (the float files, no `.Double` suffix).
- **Validation gate**: full Selig database, `tools/fortran-debug/reference/selig_passing.txt` (100,228 vectors). Required: 100% bit-exact (0 ULP in CD and CL on every vector).
- **Smoke gate** (every code change): NACA 4455/4455 set in `tools/fortran-debug/reference/clean_fortran_polar_vectors_4875.txt`, also 100% bit-exact.
- **How to run**:
  ```bash
  dotnet build tools/fortran-debug/ParallelPolarCompare/ParallelPolarCompare.csproj -c Release
  XFOIL_DISABLE_FMA=1 dotnet run --project tools/fortran-debug/ParallelPolarCompare -c Release --no-build -- \
      tools/fortran-debug/reference/selig_passing.txt --bfp
  ```

### #2 Algorithmic parity (double)

The doubled tree. *Verifiably the same code as #1*, except every `float` is widened to `double`. Auto-generated by text substitution from the float source — never hand-edited. Regenerable in one command, so any drift between #1 and #2 is detectable as a non-zero `git diff`.

- **Lives in**: `src/XFoil.Solver/Services/*.Double.cs`, `src/XFoil.Solver/Numerics/*.Double.cs`. Namespace: `XFoil.Solver.Double.Services` / `XFoil.Solver.Double.Numerics`.
- **Generator**: `tools/gen-double/gen-double.py`. Substitution rules: namespace `XFoil.X.Y` → `XFoil.X.Double.Y`, `float` → `double`, `MathF.` → `Math.`, literal `1f` → `1d`, etc.
- **Validation gate**: regenerate after any change to the float source and confirm zero diff in `*.Double.cs`. The doubled tree's facade and physical-envelope tests (`DoubledTreeFacadeTests`, `PhysicalEnvelopeTests`) must all pass.
- **How to regenerate**:
  ```bash
  python3 tools/gen-double/gen-double.py src/XFoil.Solver/Services/Foo.cs src/XFoil.Solver/Numerics/Bar.cs ...
  ```
- **Why widen**: precision floor only — same algorithm, same trajectory, fewer rounding artefacts. No new physics.

### #3 Modern

The improvement branch. Modern facade classes **inherit** from their Doubled (#2) counterparts by default; they `override` specific methods one at a time as algorithmic improvements land. Un-overridden methods inherit the Doubled implementation, so future #2 changes propagate automatically.

- **Lives in**: `src/XFoil.Solver/Services/*.Modern.cs` and `src/XFoil.Core/Services/*.Modern.cs`. Namespace: `XFoil.Solver.Modern.Services` / `XFoil.Core.Modern.Services`.
- **Architecture**: facade classes (`AirfoilAnalysisService`, `NacaAirfoilGenerator`) are `class` (not `sealed`) with `virtual` public methods in #1; gen-double preserves the modifiers, so the Doubled twin is also inheritable. `Modern.AirfoilAnalysisService : XFoil.Solver.Double.Services.AirfoilAnalysisService` — empty by default, gains an `override` per improvement.
- **Why facades only**: the static helpers (`LinearVortexInviscidSolver`, `ViscousSolverEngine`, `PolarSweepRunner`, etc.) cannot be inherited (C# disallows static-class inheritance). Modern improvements live in the facade's orchestration layer — multi-start Newton, type-3 polar fix, transition correlations all wrap or replace calls into the existing static helpers.
- **Validation gate**: external reference data (wind-tunnel measurements + 1000 OpenFOAM RANS cases with k-ω SST). Goal: better agreement with measurement than #1/#2, while #1's Fortran-bit-exact gate continues to pass.
- **Tripwire**: `ModernTreeAgreesWithDoubled_NoOverrides` asserts bit-exact equality between Modern and Doubled for every un-overridden method, so accidental drift surfaces immediately.

#### Modern overrides landed

| Tier | Override | Effect |
|------|----------|--------|
| A1   | `AnalyzeViscous` — multi-start retry on non-physical primary result (alpha jitter, ±0.05° steps, up to 5 attempts, accept first physical hit) | +63 / 5000 Selig samples converged vs Doubled (4664 vs 4601). NACA 4455 = 100% bit-exact preserved. Bit-exact-equivalent to Doubled when the primary result is already physical, so the tripwire stays green for un-jittered cases. |
| A2   | `SweepViscousRe` — fresh per-Re-point state via the facade (`AnalyzeInviscidForLiftCoefficient` -> `this.AnalyzeViscous`), instead of `PolarSweepRunner.SweepRe`'s shared inviscid state | Defense-in-depth against the historical Type-3 zero-convergence bug. On the canonical NACA 4412 fixed-CL=0.5 sweep both Modern and Doubled now converge 6/6 (the bug from the original plan was apparently fixed in float-parity by other work), so this is a no-measurable-gain override on that case. Each inner viscous call also gets the Tier A1 multi-start. |

## Validation tiers

| Tier | When | Sample |
|------|------|--------|
| Smoke gate | Every code change touching float source | NACA 4455 set (`clean_fortran_polar_vectors_4875.txt`) |
| Dev sweep | Local development | 5,000 random vectors from `selig_passing.txt` (`--sample 5000`) |
| Final/release gate | Pre-release, post-cleanup, post-merge | Full `selig_passing.txt` (100,228 vectors), 100% bit-exact required |
