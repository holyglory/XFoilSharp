## Notes

- The parity suites are part of the migration surface, not just passive regression checks. When a harness correction changes which reference packet family is authoritative, document it here and in `agents/architecture/ParityAndTodos.md`.
- Promoted-broad matrix rows now also need provenance honesty. If a row borrows valid packets from broader persisted traces, keep the owner-versus-borrowed split documented alongside the headline green/red state.
- Promoted-broad closeout also has an explicit matrix summary now. Treat the promoted-only counts and owner-gap table in the matrix output as the authoritative acceptance scoreboard for the current broad parity wave.
- Promoted broad rows are acceptance-only now. When one reopens, route the work into the earliest owner family instead of patching the broad consumer directly.
- Canonical owner promotions are tracked separately from broad promoted rows now. When a focused phase-2 rig is elevated through `canonical_phase1_promoted_rig_ids`, keep the owner role documented near the test family so the canonical phase-1 lane stays traceable to the broad row it backs.
- That canonical owner lane is larger now. The alpha-0 station-3 / station-4 seed-carry-transition chain plus `legacy-direct-seed-carry-packets` were promoted into the canonical phase-1 queue on March 22, but they still follow the same rule as the broader matrix: station numbers are witness locations inside shared code, and every `1000+` trace rig must harvest a diversified corpus across multiple airfoils / Reynolds numbers / alphas.
- `PredictedEdgeVelocityMicroParityTests` now also carries station-owned contributor facts for the alpha-0 reduced-panel upper station-4 `UESET` block. Those tests let the matrix split one broad consumer into many fast source-station owner rows so lower-tail contributor drift can be harvested and rerun independently.
- The same boundary now has an alpha-10 wake closure ladder too. `WakeColumnFortranParityTests` owns the live first-wake `PSILIN` gamma packet (`segment16->17`), `PredictedEdgeVelocityMicroParityTests` owns the upper station-2 source-lower-station-36 wake contributor, and `PreNewtonWakeUsavParityTests` owns the upper station-2 air running-sum oracle plus the late lower-wake state/remarch witnesses at stations `44` and `46`.

Those facts keep lower station-8 / station-9 only as witnesses, but the corresponding matrix parent (`legacy-direct-seed-carry-packets`) harvests all direct `laminar_seed_system`, `laminar_seed_step`, and `laminar_seed_final` packets it can find across the same diversified sweep space.
